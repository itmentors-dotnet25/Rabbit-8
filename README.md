# 🐇 Блок 8. Асинхронная коммуникация с RabbitMQ в .NET

## 🎯 Общее описание
Освойте асинхронную коммуникацию между микросервисами с использованием RabbitMQ. Научитесь проектировать надежные системы обмена сообщениями, обрабатывать ошибки и обеспечивать гарантированную доставку сообщений в распределенных приложениях.

**Стек:** .NET 8, RabbitMQ, Docker, RabbitMQ.Client, xUnit

### 🔧 Задача 2: Реализация Producer (издателя)

1. **Создайте сервис для публикации сообщений**:
   ```csharp
   public interface IMessageProducer
   {
       Task PublishAsync<T>(T message, string exchange, string routingKey);
   }
   ```
2. Реализуйте класс RabbitMQProducer, который сериализует объекты в JSON и отправляет в указанную очередь
3. Настройте базовое подключение через ConnectionFactory

### 📨 Задача 3: Реализация Consumer (потребителя)

1. **Создайте фоновый сервис для прослушивания очередей**:
   ```csharp
   public class RabbitMQConsumerService : BackgroundService
   {
       protected override async Task ExecuteAsync(CancellationToken stoppingToken)
       {
           // Подключение к RabbitMQ и подписка на очередь
       }
   }
   ```
2. Реализуйте десериализацию входящих JSON-сообщений в указанный тип T (можете создать любой класс с любыми полями)
3. Настройте подтверждение успешной обработки сообщений (BasicAck)

### 👩🏻‍💻 Задача 4. Цепочка обработки сообщений

1. Consumer принимает JSON сообщение из очереди input-queue
2. Десериализует сообщение в указанный тип, обрабатывает его
3. Передает обработанное сообщение Producer'у для публикации в очередь output-queue

### 🛡️ Задача 5: Обработка ошибок и Dead Letter Queue (DLQ)

1. Создайте DLQ для хранения неудачно обработанных сообщений
2. Настройте основную очередь для перенаправления сообщений в DLQ после N неудачных попыток обработки
3. Реализуйте базовый мониторинг сообщений в DLQ

4. **Протестируйте сценарии с ошибками**:

## 📚 Основные концепции

### RabbitMQ архитектура
- Producer/Consumer паттерн
- Exchange/Queue/Binding
- Декларативная конфигурация

### Надежность и гарантии
- Publisher Confirms
- Consumer Acknowledgements
- Сохранение сообщений на диск

### Обработка ошибок
- Dead Letter Queues (DLQ)
- Повторные попытки (Retry)
- TTL (Time-To-Live) сообщений
